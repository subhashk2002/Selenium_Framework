================================================================================
DESIGN PATTERNS USED
================================================================================

1. SINGLETON PATTERN
   → ConfigManager, DriverManager, ReportManager, AuthTokenManager
   Benefits: Single instance, centralized access, lazy initialization

2. FACTORY PATTERN
   → DriverFactory, CapabilityFactory, WaitFactory, DataReaderFactory
   Benefits: Object creation abstraction, easy extension

3. TEMPLATE METHOD PATTERN
   → BasePage, BaseTest, BaseActions, BaseAPI
   Benefits: Code reuse, extensibility, consistent flow

4. BUILDER PATTERN
   → TestDataBuilder, RequestSpecBuilder, ResponseSpecBuilder
   Benefits: Fluent API, readable code, complex object construction

5. STRATEGY PATTERN
   → DataReader interface (JSON, Excel, CSV, DB implementations)
   Benefits: Pluggable strategies, flexible data source selection

6. LISTENER PATTERN
   → ITestListener, IRetryAnalyzer, WebDriver.Listener
   Benefits: Event-driven execution, loose coupling

7. UTILITY/HELPER PATTERN
   → WaitHelper, FileUtils, RandomDataGenerator, ScreenshotUtils
   Benefits: Reusable functionality, separation of concerns
